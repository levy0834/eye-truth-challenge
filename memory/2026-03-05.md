## 🎯 20:58 飞书图片消息发送成功经验

**最终成功方案**: 卡片消息 (Card Message) + `file_path` 参数

**发送代码**:
```json
{
  "action": "send",
  "target": "user:ou_xxx",
  "message": "📸 截图",
  "card": {
    "config": { "wide_screen_mode": true },
    "elements": [
      { "tag": "img", "img_key": "filename.png", "width": "100%" },
      { "tag": "div", "text": { "content": "**数据概览**...", "tag": "lark_md" } }
    ],
    "header": { "title": { "content": "标题", "tag": "plain_text" } }
  },
  "file_path": "/path/to/image.png"
}
```

**关键要点**:
1. ✅ 使用 `file_path` 指定本地图片路径（而非base64 buffer）
2. ✅ 配合 `card` 参数创建卡片消息，图片显示在 `img` 元素中
3. ✅ `img_key` 字段值可以是任意标识（但需和file_path对应）
4. ✅ 卡片消息兼容性优于纯图片消息

**应用场景**: 以后可以在飞书对话中直接发送图表、截图、示意图等

**失败方案对比**:
- ❌ `buffer` + base64: 发送成功但对方未收到（可能大小限制）
- ✅ `file_path` + card: 完美显示

---

**经验已记牢**: 发图片用卡片消息 + file_path 最稳！
