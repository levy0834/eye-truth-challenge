# 机票价格监控调研报告

## 可行性分析

### 方案对比

| 方案 | 数据源 | 成本 | 稳定性 | 实施难度 | 国内访问 |
|------|--------|------|--------|----------|----------|
| Skyscanner API | 聚合 | 免费(有限) | 高 | 中 | ✓ |
| Kiwi.com API | 聚合 | 付费 | 高 | 中 | ✓ |
| 航司官网爬虫 | 直采 | 免费 | 低 | 高 | ✓ |
| 携程/去哪儿爬虫 | OTA | 免费 | 低 | 高 | ✓ |
| 飞猪/同程爬虫 | OTA | 免费 | 低 | 高 | ✓ |

### 推荐方案：混合策略

**主推：公开 API + 智能缓存**
- 优先使用 Skyscanner 的免费 affiliate API（每天 1000 次请求）
- 数据缓存 1 小时，减少重复请求
- 设置价格阈值触发（如低于历史平均 20%）

**备选：定向爬虫（仅监控特定航班）**
- 目标：国航/东航/南航 官网 low-fare calendar
- 频率：每小时一次，避免封 IP
- 需要处理：User-Agent 轮换、随机延迟

## 实施计划

### Phase 1: MVP（快速验证）
1. **选择数据源**：Skyscanner API（无需密钥可测试）或直接抓携程搜索页
2. **编写抓取脚本**：Python + requests + BeautifulSoup
3. **本地缓存**：存储最近7天价格到 JSON
4. **简单规则**：价格低于最近3天平均价 15%
5. **通知**：飞书文本消息

**预计时间**：2-4 小时

### Phase 2: 稳定性增强
1. **多渠道**：同时监控 2-3 个 OTA 取交叉验证
2. **异常处理**：网络失败重试、数据异常过滤
3. **去重**：同一航班重复提醒合并
4. **用户配置**：自定义阈值、航线、日期范围

**预计时间**：4-8 小时

### Phase 3: 用户体验
1. **富媒体卡片**：飞书交互卡片（价格趋势图）
2. **订阅管理**：用户可随时添加/删除监控航线
3. **历史查询**：提供价格曲线（HTML 图表）
4. **多通道**：微信/邮件备份通知

**预计时间**：8-16 小时

## 具体参数（以杭州-北京为例）

- **航线**：HGH (杭州萧山) → PKX (北京首都) / NAY (北京南苑)
- **日期窗口**：未来 7-30 天（灵活）
- **舱位**：经济舱（可扩展）
- **触发条件**：
  - 价格 ≤ 最近 7 天最低价 × 0.85（8.5折）
  - 或 价格 ≤ 历史同期最低价 × 0.9（9折）
- **检查频率**：
  - 非高峰（工作日）：4 小时一次
  - 高峰（周末/节假日）：2 小时一次
- **静默时段**：23:00-07:00 不打扰

## 风险与注意事项

⚠️ **法律/合规**
- 爬虫需遵守 robots.txt，控制频率（≥1秒/请求）
- 不要绕过付费墙或登录验证
- 仅用于个人用途，不批量转售数据

⚠️ **技术**
- 网页结构变化会导致爬虫失效（需要定期维护）
- IP 被封风险（建议使用代理池或 CDN）
- 数据延迟（OTA 价格更新可能有 5-30 分钟滞后）

⚠️ **业务**
- 价格波动是正常市场行为，避免频繁打扰
- 设置"最低价"提醒时需结合"可接受价格区间"
- 人工确认环节：建议先提供链接，用户自行判断是否购买
