# 文档排版规范

适用于所有飞书文档、日报、报告等输出内容。

## 📐 多级标题结构

- **H1**: `# 📰 标题` - 主标题，带emoji
- **H2**: `## 🎯 板块名称` - 一级板块，用emoji区分
- **H3**: `### 🔍 细分主题` - 二级标题，突出重点

## 🎨 高亮与强调

### 引用块 (Quote >)
用于【重点摘要】、【核心观点】：
```markdown
> **【重点摘要】**\n> 这里是重点内容摘要
```

### 粗体强调
关键数据、结论：`**重要信息**`

### 表情符号
每个板块标题都加上emoji，增强识别度

## 📊 表格设计

使用Markdown表格展示结构化数据：

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |

- 表头对齐用 `|---|` 分隔
- 适量列数（不超过5列）
- 数据居中对齐

## 🎯 颜色标记体系（通过emoji+文字实现）

- 🔴 **红色**：重磅消息、风险警告（用 🔴 或 ⚠️）
- 🟠 **橙色**：警示信息（用 🟡 或 ⚠️）
- 🟢 **绿色**：成功案例、积极数据（用 ✅ 或 💹）
- 🔵 **蓝色**：趋势说明、背景信息（用 ℹ️ 或 📈）
- ⚪ **灰色**：补充说明、次要信息

## ➖ 视觉分隔

- 使用 `---` 分隔不同内容区块
- 区块间距：标题下空1行，正文与表格间空1-2行
- 段落间留白，避免文字挤压

## 📝 内容组织

1. **开篇**：主标题 + 日期 + 一句话介绍
2. **核心摘要**：3-5条精华要点（用emoji列表）
3. **各分板块**：H2标题 → 内容块（引用+正文+表格）
4. **汇总模块**：趋势总结 / 明日关注
5. **结尾**：提示语 / 免责声明

## 🔗 超链接要求

**所有新闻/资讯条目必须包含原文链接：**

✅ 正确格式（Markdown）：
```markdown
1. **[【详情】](https://example.com/full-article) 新闻标题正文**
   > 新闻摘要内容...
```

✅ 效果：
1. **[【详情】](https://example.com/full-article) 新闻标题正文**
   > 新闻摘要内容...

📌 **规则**：
- 标题必须带链接，链接文字建议用"【详情】"或"阅读原文"
- 链接颜色建议统一（如蓝色 #2196F3）
- 确保链接可点击，指向原始新闻源
- 每条新闻都必须有link字段（从百度搜索API获取的url）

## 💡 可读性技巧

- **呼吸感**：避免大段密集文字，适时换行
- **视觉动线**：emoji → 标题 → 引用 → 正文 → 表格
- **重点突出**：每个自然段至少有一个强势元素（粗体/emoji/引用）
- **一致性**：相同类型的板块保持相同格式模式

## 📌 飞书文档特殊规则

⚠️ **创建-写入分离**：
1. 先 `action: "create"` → 获取 `doc_token`
2. 再 `action: "write"` → 传入完整Markdown内容
3. **不能**在create时直接传content参数

📎 **表格说明**：飞书markdown不支持表格，需要使用 `create_table_with_values` 块操作来创建表格
